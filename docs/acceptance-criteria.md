# Acceptance Criteria — Yolüstü MVP

## Əsas Qəbul Meyarları

### 1. Qeydiyyat və Giriş
- [ ] İstifadəçi qeydiyyatdan keçə bilir
- [ ] İstifadəçi daxil ola bilir
- [ ] Validasiya xətaları düzgün göstərilir
- [ ] Uğurlu əməliyyatdan sonra düzgün yönləndirmə olur

### 2. Profil
- [ ] İstifadəçi profil yarada bilir
- [ ] İstifadəçi profili redaktə edə bilir
- [ ] Reytinq və gediş sayı görünür
- [ ] Şəhər seçimi işləyir

### 3. Gediş Yaratma
- [ ] Sürücü multi-step formla gediş yarada bilir
- [ ] Bütün tələb olunan sahələr validasiya olunur
- [ ] Yer sayı 1-4 arası məhdudlaşdırılır
- [ ] Qiymət müsbət olmalıdır
- [ ] Preview addımda bütün məlumatlar düzgün görünür

### 4. Gediş Axtarışı
- [ ] Sərnişin şəhər və tarixə görə axtarış edə bilir
- [ ] Yalnız aktiv gedişlər göstərilir
- [ ] Dolu gedişlər göstərilmir
- [ ] Filtr işləyir
- [ ] Boş nəticə düzgün göstərilir

### 5. Gediş Detalları
- [ ] Bütün gediş məlumatları görünür
- [ ] Sürücü profili görünür
- [ ] Rezerv sorğusu göndərilə bilir

### 6. Rezerv Sorğusu
- [ ] Sərnişin sorğu göndərə bilir
- [ ] İstifadəçi öz gedişinə sorğu göndərə bilmir
- [ ] Artıq sorğu varsa yenisi göndərilmir
- [ ] Sorğu "pending" statusu ilə yaradılır

### 7. Rezerv Qəbul/Rədd
- [ ] Sürücü sorğunu qəbul edə bilir
- [ ] Sürücü sorğunu rədd edə bilir
- [ ] Qəbul edildikdə boş yer azalır
- [ ] Yer yoxdursa qəbul düyməsi deaktivdir

### 8. Rezerv Statusları
- [ ] Pending, accepted, rejected, cancelled, completed statusları düzgün göstərilir
- [ ] Statuslar düzgün rənglərlə fərqləndirilir
- [ ] Ləğv edilmiş accepted rezerv yerləri geri qaytarır

### 9. Rəy/Reytinq
- [ ] Yalnız tamamlanmış gediş üçün rəy yazılır
- [ ] 1-5 ulduz seçilir
- [ ] Orta reytinq düzgün yenilənir

### 10. Admin Panel
- [ ] Admin istifadəçiləri görə bilir
- [ ] Admin istifadəçi bloklaya bilir
- [ ] Admin gediş silə bilir
- [ ] Statistikalar düzgün göstərilir

### 11. Demo Ssenarisi
- [ ] Tam demo ssenarisi kritik xəta olmadan işləyir
- [ ] Bütün ekranlar arası naviqasiya işləyir
- [ ] Mobil-first dizayn düzgün görünür
